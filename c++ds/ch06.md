# 6. Queues

\index{queue}

A queue is like the line at a store: the first person in line is the first served.
The **Queue ADT** is a first-in, first-out (FIFO) collection: items are added at the back and removed from the front.
Queues model any process where order of arrival matters: print spoolers, request handling, breadth-first search, and producer-consumer pipelines.
In this chapter you will implement queues with a circular array and with a linked list, analyze their performance, and build a generic template.

## The Queue ADT

\index{queue!ADT}

A queue supports five operations:

| Operation | Description | Precondition |
|-----------|-------------|-------------|
| `enqueue(item)` | Add item to the back | none |
| `dequeue()` | Remove and return the front item | `!empty()` |
| `front()` | Return (without removing) the front item | `!empty()` |
| `empty()` | True if no items | none |
| `size()` | Number of items | none |

The defining rule: `dequeue()` always returns the *least recently enqueued* item still in the queue.

## Applications of Queues

\index{queue!applications}

- **Breadth-first search:** BFS explores neighbors level by level.
  A queue holds the nodes to visit next.
- **Print spooler:** Jobs are printed in the order they were submitted.
- **Request handling:** Web servers queue incoming requests and handle them FIFO.
- **Producer-consumer:** One thread produces work items; another consumes them in order.
- **Simulation:** Events are queued by arrival time and processed in order.

## Array-Based Queues and Circular Arrays

\index{queue!circular array}

A naïve array queue dequeues by shifting all remaining elements left --- O(n) per dequeue.
A **circular array** eliminates the shifting by using two indices (`front_` and `back_`) that wrap around when they reach the end:

```
Indices:  0     1     2     3     4     5     6     7
Data:   [  ] [  ] [Dra] [Fra] [Car] [  ] [  ] [  ]
                    ^front_            ^back_
```

When `back_` would go past the end of the array, it wraps around to 0.
This makes both `enqueue` and `dequeue` O(1).

```cpp
#include <iostream>
#include <stdexcept>
#include <string>

class CircularQueue {
public:
    explicit CircularQueue(int capacity)
        : data_{new std::string[capacity + 1]},
          capacity_{capacity + 1},  // +1 to distinguish full from empty
          front_{0},
          back_{0},
          size_{0}
    {}

    ~CircularQueue() { delete[] data_; }

    void enqueue(const std::string& item)
    {
        if (size_ == capacity_ - 1) throw std::overflow_error("queue is full");
        data_[back_] = item;
        back_ = (back_ + 1) % capacity_;
        ++size_;
    }

    std::string dequeue()
    {
        if (empty()) throw std::underflow_error("queue is empty");
        std::string value = data_[front_];
        front_ = (front_ + 1) % capacity_;
        --size_;
        return value;
    }

    const std::string& front() const
    {
        if (empty()) throw std::underflow_error("queue is empty");
        return data_[front_];
    }

    bool empty() const { return size_ == 0; }
    int size() const { return size_; }

private:
    std::string* data_;
    int capacity_;
    int front_;
    int back_;
    int size_;
};

int main()
{
    CircularQueue q{4};
    q.enqueue("Dracula");
    q.enqueue("Frankenstein");
    q.enqueue("Carmilla");

    std::cout << "Front: " << q.front() << "\n";
    std::cout << "Size: " << q.size() << "\n";

    while (!q.empty()) {
        std::cout << q.dequeue() << "\n";
    }

    return 0;
}
```

```
Front: Dracula
Size: 3
Dracula
Frankenstein
Carmilla
```

::: {.tip}
**Wut:** The circular queue uses `capacity + 1` slots to distinguish between full and empty without a separate `size_` counter.
With exactly `capacity` slots, both the empty state (`front_ == back_`) and the full state would look the same.
Using an extra slot or a size counter resolves the ambiguity.
:::

### Wrap-Around Arithmetic

The modulo operation `(index + 1) % capacity_` wraps the index around.
When `back_` is at position `capacity_ - 1` and you enqueue, it wraps to 0.
This is the key insight behind the circular array.

::: {.tip}
**Trap:** If you forget to wrap `front_` or `back_` with `% capacity_`, the index will eventually go out of bounds.
Always apply modulo after incrementing either index.
:::

## Linked List Queue Implementation

\index{queue!linked list}

A linked list queue keeps a `head_` pointer (for `dequeue`) and a `tail_` pointer (for `enqueue`).
Enqueue appends to the tail; dequeue removes from the head.

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>
#include <string>

class LinkedQueue {
public:
    void enqueue(const std::string& item)
    {
        auto node = std::make_unique<Node>(item);
        Node* raw = node.get();
        if (!tail_) {
            head_ = std::move(node);
        } else {
            tail_->next = std::move(node);
        }
        tail_ = raw;
        ++size_;
    }

    std::string dequeue()
    {
        if (empty()) throw std::underflow_error("queue is empty");
        std::string value = std::move(head_->value);
        head_ = std::move(head_->next);
        if (!head_) tail_ = nullptr;
        --size_;
        return value;
    }

    const std::string& front() const
    {
        if (empty()) throw std::underflow_error("queue is empty");
        return head_->value;
    }

    bool empty() const { return head_ == nullptr; }
    int size() const { return size_; }

private:
    struct Node {
        std::string value;
        std::unique_ptr<Node> next;
        explicit Node(const std::string& v) : value{v} {}
    };

    std::unique_ptr<Node> head_;
    Node* tail_ = nullptr;
    int size_ = 0;
};

int main()
{
    LinkedQueue q;
    q.enqueue("Dracula");
    q.enqueue("Frankenstein");
    q.enqueue("Carmilla");

    std::cout << "Front: " << q.front() << "\n";

    while (!q.empty()) {
        std::cout << q.dequeue() << "\n";
    }

    return 0;
}
```

```
Front: Dracula
Dracula
Frankenstein
Carmilla
```

All operations are O(1) and there is no capacity limit.

## Performance Analysis

\index{queue!performance}

| Operation | Circular array | Linked list |
|-----------|----------------|-------------|
| `enqueue` | O(1) | O(1) |
| `dequeue` | O(1) | O(1) |
| `front` | O(1) | O(1) |
| `empty` | O(1) | O(1) |
| `size` | O(1) | O(1) |
| **Space** | O(capacity) | O(n) |
| **Max size** | Fixed | Unlimited |
| **Cache** | Excellent | Poor |

The same trade-off as stacks: circular arrays are faster in practice due to cache locality; linked lists are more flexible.

::: {.tip}
**Tip:** The standard library provides `std::queue<T>`, backed by `std::deque` by default.
For a queue backed by a specific container, use `std::queue<T, std::list<T>>` or `std::queue<T, std::vector<T>>`.
:::

## Implementing a Queue Template in C++

\index{queue!template}

Here is a generic queue template using a circular buffer that grows when full:

```cpp
#include <algorithm>
#include <iostream>
#include <stdexcept>
#include <string>

template<typename T>
class Queue {
public:
    explicit Queue(int initial_cap = 8)
        : data_{new T[initial_cap]},
          capacity_{initial_cap},
          front_{0},
          size_{0}
    {}

    ~Queue() { delete[] data_; }

    Queue(const Queue& other)
        : data_{new T[other.capacity_]},
          capacity_{other.capacity_},
          front_{0},
          size_{other.size_}
    {
        for (int i = 0; i < other.size_; ++i) {
            data_[i] = other.data_[(other.front_ + i) % other.capacity_];
        }
    }

    Queue& operator=(Queue other)
    {
        std::swap(data_, other.data_);
        std::swap(capacity_, other.capacity_);
        std::swap(front_, other.front_);
        std::swap(size_, other.size_);
        return *this;
    }

    Queue(Queue&& other) noexcept
        : data_{other.data_},
          capacity_{other.capacity_},
          front_{other.front_},
          size_{other.size_}
    {
        other.data_ = nullptr;
        other.capacity_ = 0;
        other.front_ = 0;
        other.size_ = 0;
    }

    void enqueue(const T& item)
    {
        if (size_ == capacity_) grow();
        data_[(front_ + size_) % capacity_] = item;
        ++size_;
    }

    T dequeue()
    {
        if (empty()) throw std::underflow_error("queue is empty");
        T value = std::move(data_[front_]);
        front_ = (front_ + 1) % capacity_;
        --size_;
        return value;
    }

    const T& front() const
    {
        if (empty()) throw std::underflow_error("queue is empty");
        return data_[front_];
    }

    bool empty() const { return size_ == 0; }
    int size() const { return size_; }

private:
    void grow()
    {
        int new_cap = capacity_ * 2;
        T* new_data = new T[new_cap];
        for (int i = 0; i < size_; ++i) {
            new_data[i] = std::move(data_[(front_ + i) % capacity_]);
        }
        delete[] data_;
        data_ = new_data;
        capacity_ = new_cap;
        front_ = 0;
    }

    T* data_;
    int capacity_;
    int front_;
    int size_;
};

int main()
{
    Queue<std::string> q;
    q.enqueue("Dracula");
    q.enqueue("Frankenstein");
    q.enqueue("Carmilla");

    std::cout << "Size: " << q.size() << "\n";

    while (!q.empty()) {
        std::cout << q.dequeue() << "\n";
    }

    Queue<int> nums;
    for (int i = 0; i < 10; ++i) nums.enqueue(i);
    std::cout << "Dequeued: " << nums.dequeue() << "\n";

    return 0;
}
```

```
Size: 3
Dracula
Frankenstein
Carmilla
Dequeued: 0
```

::: {.tip}
**Tip:** When growing a circular buffer, copy elements in logical order (front to back), not physical order.
The copy loop `data_[(front_ + i) % capacity_]` walks the buffer correctly regardless of where `front_` is.
After the copy, reset `front_` to 0 so the indices are clean.
:::

## Try It: BFS with a Queue

Breadth-first search uses a queue to explore a graph level by level.
Here is a BFS on a small graph represented as an adjacency list:

```cpp
#include <iostream>
#include <queue>
#include <string>
#include <unordered_map>
#include <unordered_set>
#include <vector>

void bfs(const std::unordered_map<std::string, std::vector<std::string>>& graph,
         const std::string& start)
{
    std::queue<std::string> q;
    std::unordered_set<std::string> visited;

    q.push(start);
    visited.insert(start);

    while (!q.empty()) {
        std::string node = q.front();
        q.pop();
        std::cout << node << "\n";

        if (graph.count(node)) {
            for (const std::string& neighbor : graph.at(node)) {
                if (!visited.count(neighbor)) {
                    visited.insert(neighbor);
                    q.push(neighbor);
                }
            }
        }
    }
}

int main()
{
    std::unordered_map<std::string, std::vector<std::string>> graph = {
        {"A", {"B", "C"}},
        {"B", {"D", "E"}},
        {"C", {"F"}},
        {"D", {}},
        {"E", {"F"}},
        {"F", {}}
    };

    std::cout << "BFS from A:\n";
    bfs(graph, "A");

    return 0;
}
```

```
BFS from A:
A
B
C
D
E
F
```

Try changing the start node or modifying the graph.

## Key Points

- A **queue** is a FIFO structure: the first item enqueued is the first dequeued.
- All five operations (`enqueue`, `dequeue`, `front`, `empty`, `size`) are O(1) for both implementations.
- **Circular array:** fixed capacity (or resizable), better cache performance, no per-element overhead.
- **Linked list:** unlimited capacity, per-node pointer overhead, worse cache performance.
- A circular array uses modulo arithmetic to wrap `front_` and `back_` indices around.
- When growing a circular buffer, copy elements in logical order and reset `front_` to 0.
- Use `std::queue<T>` from the standard library in production code.
- Queues are used for BFS, print spoolers, request handling, and producer-consumer systems.
