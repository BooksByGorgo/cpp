# 0. How to Use This Book

This book picks up where *Gorgo Continuing C++* left off.
It assumes you are comfortable with templates, inheritance, polymorphism, the standard library, smart pointers, and modern C++ idioms.
If any of those topics feel shaky, revisit the relevant chapter in *Gorgo Continuing C++* before continuing.

Each chapter introduces a data structure or design topic that working C++ programmers use regularly, with explanations, code examples, and exercises.

## Tips

**Tips** call out details you need to pay special attention to.
**Traps** warn you of common mistakes.
**Wut** calls out details that are counter-intuitive --- make sure you pay attention to those.

## Function Signatures

When this book introduces a new function or method, it shows the function's **signature** and return type.
A signature is the name and parameter list --- it uniquely identifies the function.
For example:

```cpp
void insert(int index, const T& value);
```

This tells you the function is called `insert`, takes an index and a value, and returns nothing.
You do not need to understand every detail the first time you see it, but it gives you three things at a glance: what the function is called, what goes in, and what comes out.

## Code Examples

All code examples in this book compile with:

```bash
g++ -std=c++23 -Wall -Wextra -pedantic
```

Where output is shown, it appears immediately after the code block.
Every example includes the headers it needs.

## Try It

As the intro to the most amazing programming language book ever written starts out:

> The only way to learn a new programming language is by writing programs in it.

You need to write code.
At the end of most chapters is a starter program you can type in and modify to experiment.
Do not use it as an excuse to skip writing programs from scratch --- that is the only way to master the concepts.

## Exercises

Do not skip the exercises at the end of the chapters.
You can get the answer key, but do not look at it before you work out the answer yourself.
If you look at the answer key first, the concepts will not stick.
