# 4. The List ADT

\index{list ADT}

The **List ADT** is a sequence of elements where each element has a position, and the collection supports insertion, deletion, and retrieval at any position.
It is the most general linear structure, and implementing it well requires careful thought about memory management, pointer manipulation, and invariants.
In this chapter you will build array-based lists (including a growable version), singly linked lists, and doubly linked lists, then study common pitfalls and testing strategies.

## Array-Based Lists

\index{list!array-based}

An array-based list stores elements in a contiguous block of memory.
The key operations and their complexities:

| Operation | Complexity |
|-----------|-----------|
| `at(i)` | O(1) |
| `append` | O(1) amortized (O(n) on resize) |
| `insert_at(i)` | O(n) |
| `remove_at(i)` | O(n) |
| `size` | O(1) |

Here is a fixed-capacity implementation:

```cpp
#include <iostream>
#include <stdexcept>
#include <string>

class FixedList {
public:
    explicit FixedList(int capacity)
        : data_{new std::string[capacity]}, size_{0}, capacity_{capacity}
    {}

    ~FixedList() { delete[] data_; }

    void append(const std::string& v)
    {
        if (size_ == capacity_) throw std::overflow_error("list is full");
        data_[size_++] = v;
    }

    void insert_at(int pos, const std::string& v)
    {
        if (size_ == capacity_) throw std::overflow_error("list is full");
        if (pos < 0 || pos > size_) throw std::out_of_range("invalid position");
        for (int i = size_; i > pos; --i) {
            data_[i] = data_[i - 1];
        }
        data_[pos] = v;
        ++size_;
    }

    void remove_at(int pos)
    {
        if (pos < 0 || pos >= size_) throw std::out_of_range("invalid position");
        for (int i = pos; i < size_ - 1; ++i) {
            data_[i] = data_[i + 1];
        }
        --size_;
    }

    const std::string& at(int pos) const
    {
        if (pos < 0 || pos >= size_) throw std::out_of_range("invalid position");
        return data_[pos];
    }

    int size() const { return size_; }
    bool empty() const { return size_ == 0; }

private:
    std::string* data_;
    int size_;
    int capacity_;
};

int main()
{
    FixedList list{8};
    list.append("Odyssey");
    list.append("Aeneid");
    list.insert_at(1, "Iliad");

    for (int i = 0; i < list.size(); ++i) {
        std::cout << list.at(i) << "\n";
    }

    list.remove_at(0);
    std::cout << "After remove: " << list.at(0) << "\n";

    return 0;
}
```

```
Odyssey
Iliad
Aeneid
After remove: Iliad
```

### Growable Array-Based Lists

\index{list!growable array}

A fixed capacity is impractical in general.
A **growable** array list doubles its capacity when full --- the same strategy used by `std::vector`.

```cpp
#include <algorithm>
#include <iostream>
#include <stdexcept>
#include <string>

class GrowableList {
public:
    GrowableList() : data_{new std::string[4]}, size_{0}, capacity_{4} {}

    ~GrowableList() { delete[] data_; }

    GrowableList(const GrowableList& other)
        : data_{new std::string[other.capacity_]},
          size_{other.size_},
          capacity_{other.capacity_}
    {
        std::copy(other.data_, other.data_ + other.size_, data_);
    }

    GrowableList& operator=(GrowableList other)
    {
        std::swap(data_, other.data_);
        std::swap(size_, other.size_);
        std::swap(capacity_, other.capacity_);
        return *this;
    }

    GrowableList(GrowableList&& other) noexcept
        : data_{other.data_}, size_{other.size_}, capacity_{other.capacity_}
    {
        other.data_ = nullptr;
        other.size_ = 0;
        other.capacity_ = 0;
    }

    void append(const std::string& v)
    {
        if (size_ == capacity_) grow();
        data_[size_++] = v;
    }

    void insert_at(int pos, const std::string& v)
    {
        if (pos < 0 || pos > size_) throw std::out_of_range("invalid position");
        if (size_ == capacity_) grow();
        for (int i = size_; i > pos; --i) {
            data_[i] = data_[i - 1];
        }
        data_[pos] = v;
        ++size_;
    }

    void remove_at(int pos)
    {
        if (pos < 0 || pos >= size_) throw std::out_of_range("invalid position");
        for (int i = pos; i < size_ - 1; ++i) {
            data_[i] = data_[i + 1];
        }
        --size_;
    }

    const std::string& at(int pos) const
    {
        if (pos < 0 || pos >= size_) throw std::out_of_range("invalid position");
        return data_[pos];
    }

    int size() const { return size_; }
    bool empty() const { return size_ == 0; }

private:
    void grow()
    {
        int new_cap = capacity_ * 2;
        std::string* new_data = new std::string[new_cap];
        std::copy(data_, data_ + size_, new_data);
        delete[] data_;
        data_ = new_data;
        capacity_ = new_cap;
    }

    std::string* data_;
    int size_;
    int capacity_;
};

int main()
{
    GrowableList list;
    for (const std::string& t : {"Odyssey", "Iliad", "Aeneid"}) {
        list.append(t);
    }

    std::cout << "Size: " << list.size() << "\n";

    list.insert_at(1, "Theogony");
    for (int i = 0; i < list.size(); ++i) {
        std::cout << list.at(i) << "\n";
    }

    return 0;
}
```

```
Size: 3
Odyssey
Theogony
Iliad
Aeneid
```

## Singly Linked Lists

\index{linked list!singly}

A **singly linked list** consists of nodes, where each node holds a value and a pointer to the next node.
The list keeps a pointer to the **head** (first node).

```
head -> [Odyssey|next] -> [Iliad|next] -> [Aeneid|nullptr]
```

Operations and complexities:

| Operation | Complexity |
|-----------|-----------|
| `prepend` | O(1) |
| `append` (with tail pointer) | O(1) |
| `insert_after(node)` | O(1) given the node |
| `find(value)` | O(n) |
| `remove_front` | O(1) |
| `at(i)` | O(n) |

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>
#include <string>

class SinglyLinkedList {
public:
    struct Node {
        std::string value;
        std::unique_ptr<Node> next;
        explicit Node(const std::string& v) : value{v} {}
    };

    SinglyLinkedList() : head_{nullptr}, tail_{nullptr}, size_{0} {}

    void prepend(const std::string& v)
    {
        auto node = std::make_unique<Node>(v);
        if (!head_) {
            tail_ = node.get();
        }
        node->next = std::move(head_);
        head_ = std::move(node);
        ++size_;
    }

    void append(const std::string& v)
    {
        auto node = std::make_unique<Node>(v);
        Node* raw = node.get();
        if (!head_) {
            head_ = std::move(node);
        } else {
            tail_->next = std::move(node);
        }
        tail_ = raw;
        ++size_;
    }

    void remove_front()
    {
        if (!head_) throw std::underflow_error("list is empty");
        head_ = std::move(head_->next);
        if (!head_) tail_ = nullptr;
        --size_;
    }

    // Find and remove first occurrence of value
    void remove(const std::string& v)
    {
        if (!head_) throw std::invalid_argument("value not found");

        if (head_->value == v) {
            remove_front();
            return;
        }

        for (Node* p = head_.get(); p->next; p = p->next.get()) {
            if (p->next->value == v) {
                if (p->next.get() == tail_) {
                    tail_ = p;
                }
                p->next = std::move(p->next->next);
                --size_;
                return;
            }
        }
        throw std::invalid_argument("value not found");
    }

    const std::string& front() const
    {
        if (!head_) throw std::underflow_error("list is empty");
        return head_->value;
    }

    int size() const { return size_; }
    bool empty() const { return size_ == 0; }

    void print() const
    {
        for (const Node* p = head_.get(); p; p = p->next.get()) {
            std::cout << p->value << "\n";
        }
    }

private:
    std::unique_ptr<Node> head_;
    Node* tail_;
    int size_;
};

int main()
{
    SinglyLinkedList list;
    list.append("Odyssey");
    list.append("Iliad");
    list.append("Aeneid");

    std::cout << "Size: " << list.size() << "\n";
    list.print();

    list.remove("Iliad");
    std::cout << "After remove Iliad:\n";
    list.print();

    return 0;
}
```

```
Size: 3
Odyssey
Iliad
Aeneid
After remove Iliad:
Odyssey
Aeneid
```

::: {.tip}
**Trap:** Forgetting to update the `tail_` pointer when inserting or removing the last element is one of the most common linked list bugs.
Always ask: does this operation change what the last node is?
:::

## Doubly Linked Lists

\index{linked list!doubly}

A **doubly linked list** adds a `prev` pointer to each node, enabling O(1) insertion and deletion at both ends and O(1) deletion given a pointer to the node.

```
head <-> [Odyssey|prev,next] <-> [Iliad|prev,next] <-> [Aeneid|prev,next] -> tail
```

The extra pointer costs 8 bytes per node but enables bidirectional traversal and efficient tail operations.

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>
#include <string>

class DoublyLinkedList {
public:
    struct Node {
        std::string value;
        Node* prev = nullptr;
        std::unique_ptr<Node> next;
        explicit Node(const std::string& v) : value{v} {}
    };

    DoublyLinkedList() : head_{nullptr}, tail_{nullptr}, size_{0} {}

    void append(const std::string& v)
    {
        auto node = std::make_unique<Node>(v);
        node->prev = tail_;
        Node* raw = node.get();
        if (!tail_) {
            head_ = std::move(node);
        } else {
            tail_->next = std::move(node);
        }
        tail_ = raw;
        ++size_;
    }

    void prepend(const std::string& v)
    {
        auto node = std::make_unique<Node>(v);
        node->next = std::move(head_);
        if (node->next) {
            node->next->prev = node.get();
        } else {
            tail_ = node.get();
        }
        head_ = std::move(node);
        ++size_;
    }

    void remove_back()
    {
        if (!tail_) throw std::underflow_error("list is empty");

        if (tail_->prev) {
            tail_ = tail_->prev;
            tail_->next.reset();
        } else {
            head_.reset();
            tail_ = nullptr;
        }
        --size_;
    }

    const std::string& back() const
    {
        if (!tail_) throw std::underflow_error("list is empty");
        return tail_->value;
    }

    const std::string& front() const
    {
        if (!head_) throw std::underflow_error("list is empty");
        return head_->value;
    }

    int size() const { return size_; }
    bool empty() const { return size_ == 0; }

    void print() const
    {
        for (const Node* p = head_.get(); p; p = p->next.get()) {
            std::cout << p->value << "\n";
        }
    }

    void print_reverse() const
    {
        for (const Node* p = tail_; p; p = p->prev) {
            std::cout << p->value << "\n";
        }
    }

private:
    std::unique_ptr<Node> head_;
    Node* tail_;
    int size_;
};

int main()
{
    DoublyLinkedList list;
    list.append("Odyssey");
    list.append("Iliad");
    list.append("Aeneid");

    std::cout << "Forward:\n";
    list.print();

    std::cout << "Reverse:\n";
    list.print_reverse();

    list.remove_back();
    std::cout << "After remove_back:\n";
    list.print();

    return 0;
}
```

```
Forward:
Odyssey
Iliad
Aeneid
Reverse:
Aeneid
Iliad
Odyssey
After remove_back:
Odyssey
Iliad
```

## Basic Operations --- Insert, Delete, Retrieve, Traverse

\index{list!operations}

Summarizing the complexity of the core operations across implementations:

| Operation | Growable array | Singly linked | Doubly linked |
|-----------|---------------|--------------|--------------|
| `at(i)` | O(1) | O(n) | O(n) |
| `append` | O(1) amortized | O(1)* | O(1) |
| `prepend` | O(n) | O(1) | O(1) |
| `insert_at(i)` | O(n) | O(n) | O(n) |
| `remove_front` | O(n) | O(1) | O(1) |
| `remove_back` | O(1) | O(n)** | O(1) |
| `remove(node)` | O(n) | O(n) | O(1) |
| `traverse` | O(n) | O(n) | O(n) |

*Singly linked append is O(1) only with a tail pointer.

**Singly linked `remove_back` is O(n) because you must traverse to find the second-to-last node to update `tail_`.

## Implementing a Generic List in C++

\index{list!generic template}

The earlier examples used `std::string` to keep them readable.
In practice you want a list that works for any type.
Here is a generic singly linked list template:

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>
#include <string>

template<typename T>
class LinkedList {
public:
    struct Node {
        T value;
        std::unique_ptr<Node> next;
        explicit Node(const T& v) : value{v} {}
        explicit Node(T&& v) : value{std::move(v)} {}
    };

    LinkedList() : head_{nullptr}, tail_{nullptr}, size_{0} {}

    void append(const T& v)
    {
        auto node = std::make_unique<Node>(v);
        push_node(std::move(node));
    }

    void append(T&& v)
    {
        auto node = std::make_unique<Node>(std::move(v));
        push_node(std::move(node));
    }

    void prepend(const T& v)
    {
        auto node = std::make_unique<Node>(v);
        if (!head_) tail_ = node.get();
        node->next = std::move(head_);
        head_ = std::move(node);
        ++size_;
    }

    const T& front() const
    {
        if (!head_) throw std::underflow_error("empty list");
        return head_->value;
    }

    void remove_front()
    {
        if (!head_) throw std::underflow_error("empty list");
        head_ = std::move(head_->next);
        if (!head_) tail_ = nullptr;
        --size_;
    }

    int size() const { return size_; }
    bool empty() const { return size_ == 0; }

    // Iterator for range-for support
    struct Iterator {
        Node* node;
        explicit Iterator(Node* n) : node{n} {}
        T& operator*() { return node->value; }
        Iterator& operator++() { node = node->next.get(); return *this; }
        bool operator!=(const Iterator& other) const { return node != other.node; }
    };

    Iterator begin() { return Iterator{head_.get()}; }
    Iterator end() { return Iterator{nullptr}; }

private:
    void push_node(std::unique_ptr<Node> node)
    {
        Node* raw = node.get();
        if (!head_) {
            head_ = std::move(node);
        } else {
            tail_->next = std::move(node);
        }
        tail_ = raw;
        ++size_;
    }

    std::unique_ptr<Node> head_;
    Node* tail_;
    int size_;
};

int main()
{
    LinkedList<std::string> books;
    books.append("Odyssey");
    books.append("Iliad");
    books.append("Aeneid");

    for (const auto& b : books) {
        std::cout << b << "\n";
    }

    LinkedList<int> nums;
    nums.append(10);
    nums.append(20);
    nums.append(30);

    for (const auto& n : nums) {
        std::cout << n << " ";
    }
    std::cout << "\n";

    return 0;
}
```

```
Odyssey
Iliad
Aeneid
10 20 30
```

## Common Pitfalls and Testing Strategies

\index{linked list!pitfalls}
\index{linked list!testing}

Linked list implementations are notorious for subtle bugs.
Here are the most common pitfalls:

### Off-by-One in Size

Always increment or decrement `size_` in every code path that adds or removes a node.
Use `assert(size_ >= 0)` in debug builds.

### Dangling Tail Pointer

After `remove_front` on a single-element list, both `head_` and `tail_` must be null.
After `append`, `tail_` must point to the newly added node.
Failing to update `tail_` in either case leads to a dangling pointer.

::: {.tip}
**Trap:** After every insertion and deletion, ask yourself: is `head_` correct? Is `tail_` correct?
For a single-element list, `head_` and `tail_` point to the same node.
For an empty list, both are null.
:::

### Empty List Edge Cases

Test operations on:
- An empty list
- A one-element list
- A two-element list
- A large list

One-element lists are the most common source of bugs because both `head_` and `tail_` are involved in the same node.

### Testing Strategy

Write a test function that exercises every operation with all edge cases:

```cpp
#include <cassert>
#include <iostream>
#include <string>

// Include your LinkedList template here ...

void test_linked_list()
{
    LinkedList<std::string> list;

    // Empty list
    assert(list.empty());
    assert(list.size() == 0);

    // Single element
    list.append("Odyssey");
    assert(!list.empty());
    assert(list.size() == 1);
    assert(list.front() == "Odyssey");

    // Remove single element
    list.remove_front();
    assert(list.empty());
    assert(list.size() == 0);

    // Multiple elements
    list.append("Odyssey");
    list.append("Iliad");
    list.append("Aeneid");
    assert(list.size() == 3);
    assert(list.front() == "Odyssey");

    list.remove_front();
    assert(list.front() == "Iliad");
    assert(list.size() == 2);

    std::cout << "All tests passed.\n";
}

int main()
{
    test_linked_list();
    return 0;
}
```

```
All tests passed.
```

## Key Points

- **Array-based lists** offer O(1) random access and O(n) insert/delete in the middle.
- **Growable arrays** double capacity on resize for amortized O(1) append.
- **Singly linked lists** offer O(1) prepend and O(1) append (with tail pointer), O(n) access by index.
- **Doubly linked lists** add a `prev` pointer for O(1) removal given a node and bidirectional traversal.
- Always maintain the `tail_` pointer correctly; single-element lists are the most common edge case.
- Use the **Big Five** when your list class owns heap memory.
- Templates make lists generic; iterator support enables range-for loops.
- Test all edge cases: empty, one element, two elements, many elements.
