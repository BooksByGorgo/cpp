# 5. Stacks

\index{stack}

You pushed a book on a pile and later grabbed the top one --- that is a stack.
The **Stack ADT** is a last-in, first-out (LIFO) collection: the last item added is always the first removed.
Stacks are everywhere in computing: function call frames, undo history, expression parsing, depth-first search.
In this chapter you will implement stacks two ways --- array-based and linked list --- analyze their performance, build a generic template version, and use a stack to evaluate postfix expressions.

## The Stack ADT

\index{stack!ADT}

A stack supports five operations:

| Operation | Description | Precondition |
|-----------|-------------|-------------|
| `push(item)` | Add item to the top | none |
| `pop()` | Remove and return the top item | `!empty()` |
| `top()` | Return (without removing) the top item | `!empty()` |
| `empty()` | True if no items | none |
| `size()` | Number of items | none |

The defining rule: `pop()` always returns the *most recently pushed* item.

## Applications of Stacks

\index{stack!applications}

- **Function calls:** The call stack records where each function should return and what its local variables are.
  When a function is called, a frame is pushed; when it returns, the frame is popped.
- **Undo/redo:** Every action is pushed onto an undo stack.
  Undo pops the last action and reverses it.
- **Expression parsing:** Compilers use stacks to evaluate expressions and check balanced brackets.
- **Depth-first search:** DFS of a graph or tree can be implemented iteratively with a stack.
- **Backtracking algorithms:** Maze solvers and puzzle solvers push choices and pop them on dead ends.

## Array-Based Stack Implementation

\index{stack!array-based}

An array-based stack keeps a contiguous array and an index (`top_`) to the most recently pushed element.

```cpp
#include <iostream>
#include <stdexcept>
#include <string>

class ArrayStack {
public:
    explicit ArrayStack(int capacity = 64)
        : data_{new std::string[capacity]}, top_{-1}, capacity_{capacity}
    {}

    ~ArrayStack() { delete[] data_; }

    void push(const std::string& item)
    {
        if (top_ + 1 >= capacity_) throw std::overflow_error("stack overflow");
        data_[++top_] = item;
    }

    std::string pop()
    {
        if (empty()) throw std::underflow_error("stack underflow");
        return data_[top_--];
    }

    const std::string& top() const
    {
        if (empty()) throw std::underflow_error("stack is empty");
        return data_[top_];
    }

    bool empty() const { return top_ == -1; }
    int size() const { return top_ + 1; }

private:
    std::string* data_;
    int top_;
    int capacity_;
};

int main()
{
    ArrayStack s;
    s.push("Beowulf");
    s.push("Gilgamesh");
    s.push("Kalevala");

    std::cout << "Top: " << s.top() << "\n";
    std::cout << "Size: " << s.size() << "\n";

    while (!s.empty()) {
        std::cout << s.pop() << "\n";
    }

    return 0;
}
```

```
Top: Kalevala
Size: 3
Kalevala
Gilgamesh
Beowulf
```

All five operations are O(1) because array index access and increment/decrement are constant-time.

## Linked List Stack Implementation

\index{stack!linked list}

A linked list stack uses the list's head as the top of the stack.
`push` prepends a new node; `pop` removes the head.
There is no capacity limit.

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>
#include <string>

class LinkedStack {
public:
    void push(const std::string& item)
    {
        auto node = std::make_unique<Node>(item);
        node->next = std::move(top_);
        top_ = std::move(node);
        ++size_;
    }

    std::string pop()
    {
        if (empty()) throw std::underflow_error("stack underflow");
        std::string value = std::move(top_->value);
        top_ = std::move(top_->next);
        --size_;
        return value;
    }

    const std::string& top() const
    {
        if (empty()) throw std::underflow_error("stack is empty");
        return top_->value;
    }

    bool empty() const { return top_ == nullptr; }
    int size() const { return size_; }

private:
    struct Node {
        std::string value;
        std::unique_ptr<Node> next;
        explicit Node(const std::string& v) : value{v} {}
    };

    std::unique_ptr<Node> top_;
    int size_ = 0;
};

int main()
{
    LinkedStack s;
    s.push("Beowulf");
    s.push("Gilgamesh");
    s.push("Kalevala");

    std::cout << "Top: " << s.top() << "\n";

    while (!s.empty()) {
        std::cout << s.pop() << "\n";
    }

    return 0;
}
```

```
Top: Kalevala
Kalevala
Gilgamesh
Beowulf
```

All five operations are O(1).
Unlike the array stack, there is no capacity limit --- the stack grows until the system runs out of memory.

## Performance Analysis

\index{stack!performance}

| Operation | Array-based | Linked list |
|-----------|-------------|-------------|
| `push` | O(1) | O(1) |
| `pop` | O(1) | O(1) |
| `top` | O(1) | O(1) |
| `empty` | O(1) | O(1) |
| `size` | O(1) | O(1) |
| **Space** | O(capacity) | O(n) |
| **Max size** | Fixed | Unlimited |
| **Cache** | Excellent | Poor |

**When to choose which:**
- If you know the maximum size in advance, the array-based stack is simpler and faster due to cache locality.
- If the maximum size is unknown, the linked list stack avoids both overflow and over-allocation.

::: {.tip}
**Tip:** For most practical stack uses (expression parsing, DFS), `std::stack` from the standard library is the right choice.
It is backed by `std::deque` by default, but you can use a `std::vector` backend with `std::stack<T, std::vector<T>>`.
:::

## Implementing a Stack Template in C++

\index{stack!template}

The implementations above used `std::string`.
Here is a production-quality template version that handles any type:

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>
#include <string>

template<typename T>
class Stack {
public:
    void push(const T& item)
    {
        auto node = std::make_unique<Node>(item);
        node->next = std::move(top_);
        top_ = std::move(node);
        ++size_;
    }

    void push(T&& item)
    {
        auto node = std::make_unique<Node>(std::move(item));
        node->next = std::move(top_);
        top_ = std::move(node);
        ++size_;
    }

    T pop()
    {
        if (empty()) throw std::underflow_error("stack underflow");
        T value = std::move(top_->value);
        top_ = std::move(top_->next);
        --size_;
        return value;
    }

    const T& top() const
    {
        if (empty()) throw std::underflow_error("stack is empty");
        return top_->value;
    }

    bool empty() const { return top_ == nullptr; }
    int size() const { return size_; }

private:
    struct Node {
        T value;
        std::unique_ptr<Node> next;
        explicit Node(const T& v) : value{v} {}
        explicit Node(T&& v) : value{std::move(v)} {}
    };

    std::unique_ptr<Node> top_;
    int size_ = 0;
};

int main()
{
    Stack<int> ints;
    ints.push(10);
    ints.push(20);
    ints.push(30);
    std::cout << ints.pop() << " " << ints.pop() << "\n";

    Stack<std::string> books;
    books.push("Beowulf");
    books.push("Gilgamesh");
    std::cout << books.top() << "\n";

    return 0;
}
```

```
30 20
Gilgamesh
```

## Evaluation of Postfix Expressions Using Stacks

\index{postfix expression}
\index{stack!postfix evaluation}

A classical application of stacks is evaluating **postfix** (Reverse Polish Notation) expressions.
In postfix notation, the operator follows its operands:

- Infix: `(3 + 4) * 2`
- Postfix: `3 4 + 2 *`

The evaluation algorithm:

1. Read tokens left to right.
2. If the token is a number, push it.
3. If the token is an operator, pop two operands, apply the operator, and push the result.

```cpp
#include <iostream>
#include <sstream>
#include <stdexcept>
#include <string>
#include <vector>

// Simple stack backed by std::vector
template<typename T>
class VecStack {
public:
    void push(const T& v) { data_.push_back(v); }
    T pop()
    {
        if (empty()) throw std::underflow_error("stack underflow");
        T v = data_.back();
        data_.pop_back();
        return v;
    }
    bool empty() const { return data_.empty(); }
private:
    std::vector<T> data_;
};

double evaluate_postfix(const std::string& expr)
{
    VecStack<double> stack;
    std::istringstream iss{expr};
    std::string token;

    while (iss >> token) {
        if (token == "+" || token == "-" || token == "*" || token == "/") {
            double b = stack.pop();
            double a = stack.pop();
            if (token == "+") stack.push(a + b);
            else if (token == "-") stack.push(a - b);
            else if (token == "*") stack.push(a * b);
            else {
                if (b == 0) throw std::domain_error("division by zero");
                stack.push(a / b);
            }
        } else {
            stack.push(std::stod(token));
        }
    }

    if (stack.empty()) throw std::invalid_argument("empty expression");
    return stack.pop();
}

int main()
{
    // (3 + 4) * 2 = 14
    std::cout << evaluate_postfix("3 4 + 2 *") << "\n";

    // 5 1 2 + 4 * + 3 - = 14
    std::cout << evaluate_postfix("5 1 2 + 4 * + 3 -") << "\n";

    return 0;
}
```

```
14
14
```

::: {.tip}
**Wut:** The order of operands matters for subtraction and division.
When you pop two operands, the *second* pop is the *left* operand.
Pop `b` first, then `a`, and compute `a op b`.
Getting this backwards silently produces wrong answers for non-commutative operators.
:::

## Try It: Balanced Brackets Checker

A classic stack application: check whether brackets in a string are balanced.

```cpp
#include <iostream>
#include <stack>
#include <string>

bool is_balanced(const std::string& s)
{
    std::stack<char> st;
    for (char c : s) {
        if (c == '(' || c == '[' || c == '{') {
            st.push(c);
        } else if (c == ')' || c == ']' || c == '}') {
            if (st.empty()) return false;
            char open = st.top();
            st.pop();
            if ((c == ')' && open != '(') ||
                (c == ']' && open != '[') ||
                (c == '}' && open != '{')) {
                return false;
            }
        }
    }
    return st.empty();
}

int main()
{
    std::cout << std::boolalpha;
    std::cout << is_balanced("({[]})") << "\n";   // true
    std::cout << is_balanced("({[})") << "\n";    // false
    std::cout << is_balanced("((())") << "\n";    // false
    std::cout << is_balanced("") << "\n";          // true

    return 0;
}
```

```
true
false
false
true
```

Try extending it to handle `<` and `>`.

## Key Points

- A **stack** is a LIFO structure: the last item pushed is the first popped.
- All five operations (`push`, `pop`, `top`, `empty`, `size`) are O(1) for both implementations.
- **Array-based:** fixed capacity, better cache performance.
- **Linked list:** unlimited capacity, per-node allocation overhead.
- Use `std::stack` from the standard library in production code.
- Stacks are used for function calls, undo, expression parsing, DFS, and backtracking.
- **Postfix evaluation:** push numbers, pop two operands and push the result for each operator.
  The second pop is the left operand.
- **Balanced brackets:** push open brackets, pop and match on closing brackets.
