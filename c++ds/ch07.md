# 7. Heaps

\index{heap}

A heap is a tree stored in an array, cleverly arranged so the highest-priority element is always at the top.
The **Heap ADT** supports efficient insertion and efficient removal of the maximum (or minimum) element.
This makes heaps the backbone of priority queues --- a scheduling structure where tasks are served in order of importance, not order of arrival.
In this chapter you will learn how heaps work as complete binary trees, how to store them in arrays, how to insert and remove efficiently, and how to build a heap from scratch.

## The Heap ADT

\index{heap!ADT}

A **max-heap** satisfies the **heap property**: the value of every node is greater than or equal to the values of its children.
The root always holds the maximum value.
A **min-heap** satisfies the opposite: every node is less than or equal to its children, so the root holds the minimum.

The Heap ADT (max-heap version) supports:

| Operation | Description | Complexity |
|-----------|-------------|-----------|
| `insert(item)` | Add an item | O(log n) |
| `remove_max()` | Remove and return the maximum | O(log n) |
| `max()` | Return (without removing) the maximum | O(1) |
| `empty()` | True if no items | O(1) |
| `size()` | Number of items | O(1) |

## Complete Binary Trees and Heap Order

\index{complete binary tree}
\index{heap order property}

A **complete binary tree** fills levels left to right, with every level except possibly the last fully filled.
This structure is critical because it allows the tree to be stored efficiently in an array with no pointers.

For a node at index `i` (0-based):
- Left child: index `2*i + 1`
- Right child: index `2*i + 2`
- Parent: index `(i - 1) / 2`

For example, the max-heap `[90, 70, 80, 40, 60, 50, 30]` maps to:

```
           90  (index 0)
          /  \
        70    80  (indices 1, 2)
       /  \  /  \
      40  60 50  30  (indices 3, 4, 5, 6)
```

The heap property: 90 >= 70 and 80; 70 >= 40 and 60; 80 >= 50 and 30.

::: {.tip}
**Wut:** A heap is *not* a sorted array.
The heap property only guarantees a parent is >= its children --- not that all left subtree values are less than all right subtree values.
:::

## Array Representation of Heaps

\index{heap!array representation}

Because a complete binary tree fills left to right, you can store it in an array with no wasted space and no pointers:

```cpp
#include <iostream>
#include <vector>

int main()
{
    // Max-heap stored as array
    std::vector<int> heap = {90, 70, 80, 40, 60, 50, 30};

    int i = 1;  // node at index 1 (value 70)
    std::cout << "Node: " << heap[i] << "\n";
    std::cout << "Left child: " << heap[2*i + 1] << "\n";   // index 3
    std::cout << "Right child: " << heap[2*i + 2] << "\n";  // index 4
    std::cout << "Parent: " << heap[(i - 1) / 2] << "\n";   // index 0

    return 0;
}
```

```
Node: 70
Left child: 40
Right child: 60
Parent: 90
```

This array layout has excellent cache locality --- adjacent levels are adjacent in memory.

## Insertion and Removal

\index{heap!insertion}
\index{heap!removal}

### Insertion: Sift Up

To insert:
1. Append the new item at the end.
2. **Sift up**: while the new item is greater than its parent, swap them and move up.

```
Insert 85 into [90, 70, 80, 40, 60, 50, 30]:
  Append: [90, 70, 80, 40, 60, 50, 30, 85]  (index 7)
  85 > parent(index 3 = 40)? Yes, swap:
    [90, 70, 80, 85, 60, 50, 30, 40]
  85 > parent(index 1 = 70)? Yes, swap:
    [90, 85, 80, 70, 60, 50, 30, 40]
  85 > parent(index 0 = 90)? No, stop.
```

### Removal: Sift Down

To remove the maximum:
1. Swap the root with the last element.
2. Remove the last element (the old root, now at the end).
3. **Sift down**: while the new root is less than either child, swap it with the larger child and move down.

```
Remove max from [90, 85, 80, 70, 60, 50, 30, 40]:
  Swap root with last: [40, 85, 80, 70, 60, 50, 30, 90]
  Remove last: [40, 85, 80, 70, 60, 50, 30]
  Sift down 40:
    40 < max(85, 80) = 85, swap with index 1:
      [85, 40, 80, 70, 60, 50, 30]
    40 < max(70, 60) = 70, swap with index 3:
      [85, 70, 80, 40, 60, 50, 30]
    40 has no children, stop.
```

Both operations touch at most O(log n) nodes --- the height of the tree.

## Heap Construction

\index{heap!construction}

You can build a heap from an arbitrary array in O(n) time using the **heapify** algorithm:
start from the last internal node (index `(n-1)/2`) and sift down each node in reverse order.

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

void sift_down(std::vector<int>& heap, int i, int n)
{
    while (true) {
        int largest = i;
        int left = 2 * i + 1;
        int right = 2 * i + 2;

        if (left < n && heap[left] > heap[largest]) largest = left;
        if (right < n && heap[right] > heap[largest]) largest = right;

        if (largest == i) break;
        std::swap(heap[i], heap[largest]);
        i = largest;
    }
}

void build_heap(std::vector<int>& arr)
{
    int n = static_cast<int>(arr.size());
    for (int i = (n - 1) / 2; i >= 0; --i) {
        sift_down(arr, i, n);
    }
}

int main()
{
    std::vector<int> arr = {3, 1, 4, 1, 5, 9, 2, 6};
    build_heap(arr);

    std::cout << "Heap: ";
    for (int v : arr) {
        std::cout << v << " ";
    }
    std::cout << "\n";
    std::cout << "Max: " << arr[0] << "\n";

    return 0;
}
```

```
Heap: 9 6 4 3 5 1 2 1
Max: 9
```

::: {.tip}
**Wut:** Building a heap with `n` repeated insertions costs O(n log n).
The bottom-up `build_heap` algorithm costs O(n).
The key insight is that most nodes are near the leaves, where sift-down is very short.
:::

## Priority Queues and Heaps

\index{priority queue}

A **priority queue** is an ADT that always gives you the highest-priority element next.
A heap is the natural implementation: the max (or min) is always at the root.

```cpp
#include <iostream>
#include <queue>
#include <string>

int main()
{
    // Max-priority queue (largest priority first)
    std::priority_queue<int> pq;
    pq.push(3);
    pq.push(7);
    pq.push(1);
    pq.push(9);
    pq.push(5);

    while (!pq.empty()) {
        std::cout << pq.top() << " ";
        pq.pop();
    }
    std::cout << "\n";

    return 0;
}
```

```
9 7 5 3 1
```

For a min-priority queue, use `std::priority_queue<T, std::vector<T>, std::greater<T>>`.

### Task Scheduling Example

```cpp
#include <iostream>
#include <queue>
#include <string>
#include <utility>

int main()
{
    // Priority queue of (priority, task_name) --- higher priority runs first
    using Task = std::pair<int, std::string>;
    std::priority_queue<Task> tasks;

    tasks.push({3, "Emma"});
    tasks.push({9, "Persuasion"});
    tasks.push({5, "Mansfield Park"});
    tasks.push({1, "Northanger"});

    std::cout << "Processing tasks by priority:\n";
    while (!tasks.empty()) {
        auto [priority, name] = tasks.top();
        tasks.pop();
        std::cout << "  [" << priority << "] " << name << "\n";
    }

    return 0;
}
```

```
Processing tasks by priority:
  [9] Persuasion
  [5] Mansfield Park
  [3] Emma
  [1] Northanger
```

## Implementing a Binary Heap in C++

\index{heap!implementation}

Here is a complete generic max-heap template:

```cpp
#include <algorithm>
#include <iostream>
#include <stdexcept>
#include <vector>

template<typename T>
class MaxHeap {
public:
    void insert(const T& item)
    {
        data_.push_back(item);
        sift_up(static_cast<int>(data_.size()) - 1);
    }

    T remove_max()
    {
        if (empty()) throw std::underflow_error("heap is empty");
        T max = data_[0];
        data_[0] = std::move(data_.back());
        data_.pop_back();
        if (!data_.empty()) sift_down(0);
        return max;
    }

    const T& max() const
    {
        if (empty()) throw std::underflow_error("heap is empty");
        return data_[0];
    }

    bool empty() const { return data_.empty(); }
    int size() const { return static_cast<int>(data_.size()); }

private:
    void sift_up(int i)
    {
        while (i > 0) {
            int parent = (i - 1) / 2;
            if (data_[i] <= data_[parent]) break;
            std::swap(data_[i], data_[parent]);
            i = parent;
        }
    }

    void sift_down(int i)
    {
        int n = static_cast<int>(data_.size());
        while (true) {
            int largest = i;
            int left = 2 * i + 1;
            int right = 2 * i + 2;
            if (left < n && data_[left] > data_[largest]) largest = left;
            if (right < n && data_[right] > data_[largest]) largest = right;
            if (largest == i) break;
            std::swap(data_[i], data_[largest]);
            i = largest;
        }
    }

    std::vector<T> data_;
};

int main()
{
    MaxHeap<int> heap;
    for (int v : {5, 3, 9, 1, 7, 4, 8}) {
        heap.insert(v);
    }

    std::cout << "Heap sort: ";
    while (!heap.empty()) {
        std::cout << heap.remove_max() << " ";
    }
    std::cout << "\n";

    return 0;
}
```

```
Heap sort: 9 8 7 5 4 3 1
```

::: {.tip}
**Tip:** Removing elements from a max-heap in order produces them sorted from largest to smallest.
This is the basis of **Heap Sort**, an O(n log n) in-place sorting algorithm.
:::

## Performance Analysis

\index{heap!performance}

| Operation | Complexity |
|-----------|-----------|
| `insert` | O(log n) |
| `remove_max` | O(log n) |
| `max` | O(1) |
| `empty` | O(1) |
| `size` | O(1) |
| `build_heap` | O(n) |
| **Space** | O(n) |

The O(log n) operations come from the height of the complete binary tree, which is ⌊log₂ n⌋.

## Try It: Heap Sort

Use the `MaxHeap` template above to sort a list of integers.
Then compare it to `std::sort` and verify they produce the same result.

```cpp
#include <algorithm>
#include <cassert>
#include <iostream>
#include <vector>

// Assume MaxHeap is defined above

int main()
{
    std::vector<int> input = {5, 3, 9, 1, 7, 4, 8, 2, 6};

    // Sort using heap
    MaxHeap<int> heap;
    for (int v : input) heap.insert(v);

    std::vector<int> heap_sorted;
    while (!heap.empty()) {
        heap_sorted.push_back(heap.remove_max());
    }
    // heap_sorted is descending; reverse for ascending
    std::reverse(heap_sorted.begin(), heap_sorted.end());

    // Sort using std::sort
    std::vector<int> std_sorted = input;
    std::sort(std_sorted.begin(), std_sorted.end());

    assert(heap_sorted == std_sorted);
    std::cout << "Sorted: ";
    for (int v : heap_sorted) std::cout << v << " ";
    std::cout << "\n";

    return 0;
}
```

```
Sorted: 1 2 3 4 5 6 7 8 9
```

## Key Points

- A **heap** is a complete binary tree stored in an array with the heap order property.
- **Max-heap**: every parent >= its children; root holds the maximum.
- **Min-heap**: every parent <= its children; root holds the minimum.
- Array index formulas: left child at `2i+1`, right child at `2i+2`, parent at `(i-1)/2`.
- **Insert**: append, then sift up --- O(log n).
- **Remove max**: swap root with last, remove last, sift down --- O(log n).
- **Max**: read `data_[0]` --- O(1).
- **Build heap** (bottom-up): O(n), faster than n repeated insertions (O(n log n)).
- A **priority queue** backed by a heap always gives the highest-priority element next.
- Heap sort is O(n log n) and uses O(1) extra space.
