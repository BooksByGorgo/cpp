# 11. Reusability Through Templates

\index{template!reusability}

You have been writing template data structures since Chapter 2, but templates have more power than you have used so far.
Generic data structures can be made more flexible and safer with template constraints, default template parameters, and non-type template parameters.
This chapter deepens your understanding of templates as applied to data structures: how to constrain types, how to provide sensible defaults, and how to write generic algorithms that work with your containers.

## Template Constraints with Concepts

\index{concept}
\index{template!concept}

Templates accept any type by default, which can produce cryptic error messages when a type does not support the required operations.
C++20 **concepts** let you express constraints clearly and generate meaningful compiler errors.

```cpp
#include <concepts>
#include <iostream>
#include <string>

// A type must support < and == to be stored in a sorted structure
template<typename T>
concept Comparable = requires(T a, T b) {
    { a < b } -> std::convertible_to<bool>;
    { a == b } -> std::convertible_to<bool>;
};

template<Comparable T>
class SortedArray {
public:
    void insert(const T& value)
    {
        int pos = 0;
        while (pos < size_ && data_[pos] < value) ++pos;
        for (int i = size_; i > pos; --i) data_[i] = data_[i - 1];
        data_[pos] = value;
        ++size_;
    }

    void print() const
    {
        for (int i = 0; i < size_; ++i) {
            std::cout << data_[i] << "\n";
        }
    }

private:
    T data_[64]{};
    int size_ = 0;
};

int main()
{
    SortedArray<std::string> arr;
    arr.insert("Purgatorio");
    arr.insert("Inferno");
    arr.insert("Paradiso");
    arr.print();

    return 0;
}
```

```
Inferno
Paradiso
Purgatorio
```

If you try to instantiate `SortedArray<SomeType>` where `SomeType` does not support `<`, the compiler gives a clear error: "constraint not satisfied."
Without concepts, the error would point deep inside the template body.

::: {.tip}
**Tip:** Use concepts to document which operations a template requires.
Even before C++20, you can write a comment listing the requirements.
With concepts, the documentation becomes machine-checked.
:::

## Default Template Parameters

\index{template!default parameters}

You can provide default types for template parameters.
This lets users customize behavior without having to specify types they do not care about:

```cpp
#include <functional>
#include <iostream>
#include <string>

// A priority queue with a customizable comparator
template<typename T, typename Compare = std::less<T>>
class PriorityQueue {
public:
    void push(const T& v)
    {
        data_[size_++] = v;
        sift_up(size_ - 1);
    }

    T pop()
    {
        T top = data_[0];
        data_[0] = data_[--size_];
        sift_down(0);
        return top;
    }

    const T& top() const { return data_[0]; }
    bool empty() const { return size_ == 0; }

private:
    void sift_up(int i)
    {
        while (i > 0) {
            int p = (i - 1) / 2;
            if (!cmp_(data_[p], data_[i])) break;
            std::swap(data_[i], data_[p]);
            i = p;
        }
    }

    void sift_down(int i)
    {
        while (true) {
            int best = i;
            int l = 2 * i + 1, r = 2 * i + 2;
            if (l < size_ && cmp_(data_[best], data_[l])) best = l;
            if (r < size_ && cmp_(data_[best], data_[r])) best = r;
            if (best == i) break;
            std::swap(data_[i], data_[best]);
            i = best;
        }
    }

    T data_[64]{};
    int size_ = 0;
    Compare cmp_{};
};

int main()
{
    // Default: max-heap (std::less gives max at top)
    PriorityQueue<int> max_pq;
    for (int v : {3, 1, 4, 1, 5, 9, 2, 6}) max_pq.push(v);
    std::cout << "Max: " << max_pq.top() << "\n";

    // Min-heap: use std::greater
    PriorityQueue<int, std::greater<int>> min_pq;
    for (int v : {3, 1, 4, 1, 5, 9, 2, 6}) min_pq.push(v);
    std::cout << "Min: " << min_pq.top() << "\n";

    return 0;
}
```

```
Max: 9
Min: 1
```

The caller uses the default `PriorityQueue<int>` for the common case (max-heap) and only specifies the comparator when customization is needed.

## Non-Type Template Parameters

\index{template!non-type parameters}

Template parameters do not have to be types --- they can be compile-time integer values.
This lets you create fixed-size containers where the size is a compile-time constant:

```cpp
#include <cassert>
#include <iostream>
#include <stdexcept>
#include <string>

template<typename T, int N>
class FixedStack {
public:
    static_assert(N > 0, "stack capacity must be positive");

    void push(const T& v)
    {
        if (top_ >= N) throw std::overflow_error("stack overflow");
        data_[top_++] = v;
    }

    T pop()
    {
        if (empty()) throw std::underflow_error("stack underflow");
        return data_[--top_];
    }

    const T& top() const
    {
        if (empty()) throw std::underflow_error("stack is empty");
        return data_[top_ - 1];
    }

    bool empty() const { return top_ == 0; }
    int size() const { return top_; }
    static constexpr int capacity() { return N; }

private:
    T data_[N];
    int top_ = 0;
};

int main()
{
    FixedStack<std::string, 4> s;
    s.push("Inferno");
    s.push("Purgatorio");
    s.push("Paradiso");

    std::cout << "Capacity: " << s.capacity() << "\n";
    std::cout << "Size: " << s.size() << "\n";

    while (!s.empty()) {
        std::cout << s.pop() << "\n";
    }

    return 0;
}
```

```
Capacity: 4
Size: 3
Paradiso
Purgatorio
Inferno
```

`FixedStack<std::string, 4>` and `FixedStack<std::string, 8>` are *different types* --- each has its own specialization with a different array size baked in.
The `static_assert` catches invalid sizes at compile time.

## Generic Algorithms for Your Containers

\index{generic algorithm}

Template algorithms work with any type that satisfies certain constraints.
You can write generic algorithms for your own containers by parameterizing on iterators:

```cpp
#include <iostream>
#include <string>

// Generic print: works with any container that has begin() and end()
template<typename Container>
void print_all(const Container& c)
{
    for (const auto& item : c) {
        std::cout << item << "\n";
    }
}

// Generic count_if: counts elements satisfying a predicate
template<typename Container, typename Predicate>
int count_if(const Container& c, Predicate pred)
{
    int count = 0;
    for (const auto& item : c) {
        if (pred(item)) ++count;
    }
    return count;
}

int main()
{
    LinkedList<std::string> books;
    books.append("Inferno");
    books.append("Purgatorio");
    books.append("Paradiso");

    print_all(books);

    int long_titles = count_if(books,
        [](const std::string& s) { return s.size() > 7; });
    std::cout << "Titles longer than 7 chars: " << long_titles << "\n";

    return 0;
}
```

This is exactly how the standard library algorithms work: they are parameterized on iterators, and any container that provides compatible iterators can use them.

## Template Specialization

\index{template!specialization}

Sometimes you need different behavior for a specific type.
**Template specialization** lets you provide a custom implementation for one type while the primary template handles everything else:

```cpp
#include <algorithm>
#include <cstring>
#include <iostream>
#include <string>

// Primary template: generic equality comparison
template<typename T>
bool equal(const T& a, const T& b)
{
    return a == b;
}

// Specialization for const char*: use strcmp instead of pointer comparison
template<>
bool equal<const char*>(const char* const& a, const char* const& b)
{
    return std::strcmp(a, b) == 0;
}

int main()
{
    std::cout << std::boolalpha;
    std::cout << equal(42, 42) << "\n";           // uses primary template
    std::cout << equal(3.14, 3.14) << "\n";       // uses primary template

    const char* s1 = "hello";
    const char* s2 = "hello";
    std::cout << equal(s1, s2) << "\n";            // uses specialization

    return 0;
}
```

```
true
true
true
```

Without the specialization, `equal(s1, s2)` would compare pointer addresses, not string content.

::: {.tip}
**Wut:** Template specializations do not participate in overload resolution the same way regular functions do.
For most cases, function overloads or `if constexpr` (C++17) are better alternatives to function template specializations.
:::

## Try It: A Generic Ring Buffer

Combine non-type template parameters with concepts to build a generic ring buffer:

```cpp
#include <concepts>
#include <iostream>
#include <stdexcept>
#include <string>

template<typename T, int N>
requires (N > 0)
class RingBuffer {
public:
    void push(const T& v)
    {
        if (full()) throw std::overflow_error("ring buffer full");
        data_[(head_ + size_) % N] = v;
        ++size_;
    }

    T pop()
    {
        if (empty()) throw std::underflow_error("ring buffer empty");
        T v = data_[head_];
        head_ = (head_ + 1) % N;
        --size_;
        return v;
    }

    bool empty() const { return size_ == 0; }
    bool full() const { return size_ == N; }
    int size() const { return size_; }

private:
    T data_[N]{};
    int head_ = 0;
    int size_ = 0;
};

int main()
{
    RingBuffer<std::string, 3> buf;
    buf.push("Inferno");
    buf.push("Purgatorio");
    buf.push("Paradiso");

    std::cout << "Full: " << buf.full() << "\n";
    std::cout << buf.pop() << "\n";
    std::cout << buf.pop() << "\n";
    buf.push("Vita Nuova");
    std::cout << "Size: " << buf.size() << "\n";

    return 0;
}
```

```
Full: 1
Inferno
Purgatorio
Size: 2
```

## Key Points

- **Concepts** (C++20) let you constrain template parameters and produce clear error messages.
- **Default template parameters** let users customize behavior without specifying parameters they do not need.
- **Non-type template parameters** allow compile-time constant values (like array sizes) as template arguments.
- Different template instantiations (e.g., `FixedStack<T, 4>` vs. `FixedStack<T, 8>`) are separate types.
- **Generic algorithms** parameterized on iterators work with any container that provides compatible iterators.
- **Template specialization** provides custom behavior for specific types; prefer `if constexpr` for conditional logic within a template.
- Standard library containers and algorithms are templates for exactly this reason: zero overhead abstraction.
